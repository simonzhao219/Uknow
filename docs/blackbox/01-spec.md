## 1. 產品概觀

### 1.1 產品定位
Uknow 是一款**線上多人即時對戰的 UNO 類卡牌遊戲**。使用者透過瀏覽器（Web）進入遊戲，建立或加入房間，與其他玩家（真人或電腦 AI）進行回合制出牌對戰，最先出完手牌者獲勝。

### 1.2 目標使用者
- 想與朋友線上同樂的休閒玩家
- 想快速配對陌生人對戰的一般玩家
- **[假設]** 觀戰者（可能允許旁觀不參與）

### 1.3 核心價值
- 免安裝、開瀏覽器即玩
- 即時同步的多人互動
- 規則清楚、上手快速

### 1.4 系統邊界（黑箱推定）
- **前端**：瀏覽器單頁應用（SPA）。
- **後端**：**[假設]** 具權威伺服器負責發牌、規則裁決、狀態同步（可能經 WebSocket 即時通訊）。所有規則正確性與防作弊須由伺服器端裁決，前端僅呈現。
- **[假設]** 觀測到 PR 環境接有 Supabase 整合，後端資料/驗證可能建於其上（Phase 3 確認）。

## 2. 名詞定義（Glossary）

| 名詞 | 定義 |
|---|---|
| 玩家 (Player) | 參與遊戲的使用者，可為真人或 AI |
| 觀戰者 (Spectator) | 旁觀不出牌的使用者 **[假設]** |
| 房間 (Room) | 一場遊戲的容器，玩家在此聚集並開始對局 |

## 3. 卡牌與遊戲規則（黑箱推定）

> UNO 為市場上廣為人知之標準規則，以下為推定之標準玩法。**[假設]** 實際卡牌組成、數量與特殊規則以真實系統為準。

### 3.1 卡牌種類
- **數字牌**：紅、黃、綠、藍四種花色，數字 0–9。
- **功能牌（各花色）**：跳過 (Skip)、反轉 (Reverse)、抽二 (Draw Two / +2)。
- **萬用牌（無花色）**：變色 (Wild)、變色抽四 (Wild Draw Four / +4)。

### 3.2 出牌合法性規則
- 打出的牌必須符合下列任一條件才合法：
  1. 與當前有效花色**同花色**；
  2. 與頂牌**同數字**；
  3. 與頂牌**同功能**（如 Skip 疊 Skip）；
  4. 為**萬用牌**（Wild 任何情況皆可出；+4 **[待定 D6]** 可能限「手中無同色牌才可出」）。
- 若無合法牌可出，玩家須從抽牌堆抽 1 張。抽後行為見 §3.6。

### 3.3 回合與方向
- 玩家依座位順序輪流，方向可被 Reverse 反轉。
- **[待定 D3]** 2 人局中：Reverse 是否等同 Skip、Skip/+2 後回合是否回到出牌者本人。

### 3.4 起始翻牌（**[待定 D4]**）
- 開局翻開一張作為棄牌堆頂牌。若翻到功能牌 / 萬用牌，首位玩家之處理（誰先出、效果是否即時生效、Wild 由誰選色 / 或重翻）依實際規則，Phase 3 敲定。

### 3.5 疊牌 stacking（**[待定 D5]**）
- 被 +2 / +4 的玩家能否以同類牌疊加轉嫁，或必須直接抽牌並被跳過，Phase 3 敲定。

### 3.6 抽牌後行為（**[待定 D2]**）
- 抽 1 張後，玩家可：(a) 立即打出剛抽到的合法牌；或 (b) 選擇不出、以「Pass / 結束回合」主動換手。實際採哪種、是否強制出、是否可無限抽，Phase 3 敲定。

### 3.7 勝負與計分（**[待定 D1]**）
- 單局：最先出完手牌者贏得該局。
- 賽局：**[假設]** 可能為單局定勝負，或採計分制（他人手牌點數加總給勝者，達目標分數者為最終贏家）。實際賽制 Phase 3 敲定。

### 3.8 UNO 宣告與罰則（**[待定 D7]**）
- 手牌剩 1 張時須宣告「UNO」。合法宣告時窗（出該張當下／下次輪到前）、未宣告被抓之罰則張數、誤宣告（手牌 >1 亂喊）、誣告（對已宣告者抓包）之處理，Phase 3 敲定。

### 3.9 牌堆耗盡重洗
- 抽牌堆耗盡時，將棄牌堆（保留頂牌）洗回抽牌堆。**[待定 D8]** 若可洗回牌數仍不足需抽張數（如 +4 需 4 張僅剩 2）之處理，Phase 3 敲定。重洗須：總牌數守恆、頂牌保留、順序隨機。

## 4. 功能需求（Functional Requirements）

### 4.1 使用者身分與進入
- **FR-1.1** 使用者可設定顯示暱稱後進入遊戲。**[假設]** 訪客即玩或需登入。
- **FR-1.2** **[假設]** 支援帳號註冊 / 登入。
- **FR-1.3** 暱稱有長度與字元限制；空白 / 純空白 / 過長 / 危險字元應被適當處理；**[待定]** 同房內同名是否允許。

### 4.2 大廳與房間管理
- **FR-2.1** 使用者可**建立房間**，取得可分享的房號 / 連結。房號應具足夠亂數強度、不可被列舉猜測。
- **FR-2.2** 使用者可**加入房間**：透過房號、連結或清單。房號格式非法 / 查無 / 已滿 / 已開局 皆有對應回應。
- **FR-2.3** 房間有**人數上限**（**[待定 D9]** 2–4 或其他）與**最低開局人數**（**[待定 D9]** 至少 2）。
- **FR-2.4** 房主可設定房間參數：**[假設]** 公開/私人、密碼、AI 補位、規則變體。私人房不出現在公開清單；密碼錯誤被拒且不洩漏房間內部資訊。
- **FR-2.5** 房主可**開始遊戲**；達最低人數才可開始；僅房主有此權限（伺服器端強制，非僅隱藏按鈕）。
- **FR-2.6** 玩家可**離開房間**；房主離開（含斷線）時應有房主轉移或房間關閉機制，並通知其餘玩家。
- **FR-2.7** **[假設]** 可加入 AI 玩家補足人數；AI 與真人受同一人數上限約束。
- **FR-2.8** 等待區應顯示玩家列表、房主標記；**[假設]** 房主可踢人、玩家可切換 ready 狀態。

### 4.3 遊戲進行
- **FR-3.1** 開局為每位玩家發初始手牌（**[待定 D9]** 假設 7 張），並翻開起始頂牌（見 §3.4）。
- **FR-3.2** 系統顯示：自己的手牌、各對手剩餘張數、棄牌堆頂牌、**當前有效花色**（含出 Wild 後所選色的持續指示）、出牌方向、輪到誰。
- **FR-3.3** 輪到自己時可打出**合法**手牌；不合法的牌應被拒或不可點選，並提示**具體**原因。
- **FR-3.4** 打出萬用牌時須跳出**選色介面**，選色前不換手；選色值僅限四花色。
- **FR-3.5** 無牌可出時可**抽牌**；抽後依 §3.6（**[待定 D2]**）處理；不可於一回合內越權多次抽牌。
- **FR-3.6** 功能牌效果即時生效並同步所有玩家（Skip / Reverse / +2 / +4，含 2 人局特例 §3.3）。
- **FR-3.7** 手牌剩 1 張時提供 **UNO 宣告**操作（見 §3.8）。
- **FR-3.8** 有玩家出完手牌時顯示**結算/勝利畫面**（含計分，若採計分制）。
- **FR-3.9** 局結束後可**再來一局 / 返回大廳**；再戰時人數不足 / 玩家已離開之處理需明確。
- **FR-3.10** 關鍵操作進行中（加入、開始、出牌、抽牌）應有載入指示並禁止重複提交（防連點雙送）。

### 4.4 即時同步與連線
- **FR-4.1** 所有玩家狀態即時一致；狀態更新應可容忍亂序 / 重複（以版本號或序號收斂），出牌請求應冪等（重送不重複生效）。
- **FR-4.2** 玩家斷線 / 重整後應能重連回當前對局，且手牌、頂牌、有效花色、方向、各家張數、當前回合與其他玩家一致；重連期間手牌不外洩。
- **FR-4.3** **[假設]** 玩家逾時未操作之處理（自動抽牌換手 / 連續逾時轉 AI 或踢出），Phase 3 敲定門檻。
- **FR-4.4** 弱網 / 丟包 / 伺服器 5xx 下，前端不重複扣牌、提供可理解錯誤訊息並可安全重試。

### 4.5 互動與輔助
- **FR-5.1** **[假設]** 房內聊天室 / 表情；輸入須防注入。
- **FR-5.2** **[假設]** 出牌歷史 / 操作紀錄；長對局下不應無限增長致資源洩漏。
- **FR-5.3** 非法操作提示應**具體**（如「此牌需與紅色或 5 相符」），並可在地化。

### 4.6 觀戰（**[假設]**，**[待定 D10]**）
- **FR-6.1** 若支援觀戰：觀戰者可進入進行中對局旁觀，取得正確快照，但**不得**收到任何玩家手牌牌面、**不得**出牌。
- **FR-6.2** 若不支援觀戰：規格應明確聲明不支援（Phase 3 擇一定案）。

## 6. 使用者流程（User Flows）

### 6.1 主流程：建立房間並對戰
1. 進入首頁 → 設定暱稱
2. 建立房間 → 取得房號/連結
3. 等待其他玩家加入（或加 AI）；房主可踢人，玩家可 ready
4. 房主開始遊戲 → 發牌 + 起始翻牌
5. 依回合輪流出牌 / 抽牌（抽後可出或 Pass）/ 用功能牌 / 選色 / 宣告 UNO
6. 有人出完 → 結算（計分）
7. 再來一局 或 返回大廳

### 6.2 次流程：加入既有房間
1. 設定暱稱 → 輸入房號 / 邀請連結 / 清單點選
2. 進入等待區 → 開始後參與；若對局進行中則被拒或以觀戰進入（**[待定 D10]**）

## 7. 邊界與例外情境（黑箱推定）

- 房間已滿 / 已開局 / 查無 / 房號格式非法 → 各有對應提示，房內狀態不受影響。
- 密碼房密碼錯誤 → 拒絕，不洩漏內部資訊。
- 人數不足硬開局、非房主開局 → 阻止（伺服器端）。
- 非自己回合出牌、出非法牌、出手中沒有的牌、重複出已離手的牌 → 拒絕，狀態守恆。
- 連點出牌 / 抽牌 / 開始 / 再來一局 → 僅生效一次。
- Wild 選非法色值 → 拒絕，不換手。
- UNO 誤宣告 / 誣告抓包 → 依規則無效或受罰。
- 抽牌堆耗盡 / 與棄牌堆同時近耗盡 → 重洗，牌數守恆（§3.9，**[待定 D8]**）。
- 對局中房主離開/斷線、房間中途解散 → 轉移 / 關閉 + 通知 + 導回。
- 斷線重連 → 狀態一致還原、回合與計時延續、手牌不外洩。
- 逾時 / 連續逾時 → 自動換手 / 升級處理（**[假設]**）。
- 近同時的競態操作（同時出最後一張、方向反轉瞬間雙出、同時抽最後一張）→ 伺服器一致仲裁，僅一方生效，牌數守恆。
