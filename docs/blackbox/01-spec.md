## 1. 產品概觀

### 1.1 產品定位
Uknow 是一款**線上多人即時對戰的 UNO 類卡牌遊戲**。使用者透過瀏覽器（Web）進入遊戲，建立或加入房間，與其他玩家（真人或電腦 AI）進行回合制出牌對戰，最先出完手牌者獲勝。

### 1.2 目標使用者
- 想與朋友線上同樂的休閒玩家
- 想快速配對陌生人對戰的一般玩家
- **[假設]** 觀戰者（可能允許旁觀不參與）

### 1.3 核心價值
- 免安裝、開瀏覽器即玩
- 即時同步的多人互動
- 規則清楚、上手快速

## 2. 名詞定義（Glossary）

| 名詞 | 定義 |
|---|---|
| 玩家 (Player) | 參與遊戲的使用者，可為真人或 AI |
| 房間 (Room) | 一場遊戲的容器，玩家在此聚集並開始對局 |
| 房主 (Host) | 建立房間或擁有房間控制權的玩家 |
| 大廳 (Lobby) | 顯示可加入房間清單或提供配對入口的頁面 |
| 手牌 (Hand) | 玩家目前持有、尚未打出的卡牌 |
| 牌堆 (Draw Pile) | 尚未發出、供抽牌用的卡牌堆 |
| 棄牌堆 (Discard Pile) | 已打出卡牌堆疊，頂牌決定當前可出的花色/數字 |
| 回合 (Turn) | 輪到某位玩家出牌或抽牌的時段 |
| 出牌方向 | 順時針 / 逆時針，可被特殊牌反轉 |
| UNO 宣告 | 手牌剩 1 張時須宣告「UNO」，否則受罰 |

## 3. 卡牌與遊戲規則（黑箱推定）

> UNO 為市場上廣為人知之標準規則，以下為推定之標準玩法。**[假設]** 實際卡牌組成、數量與特殊規則以真實系統為準。

### 3.1 卡牌種類
- **數字牌**：紅、黃、綠、藍四種花色，數字 0–9。
- **功能牌（各花色）**：
  - 跳過 (Skip)：下一位玩家被跳過。
  - 反轉 (Reverse)：出牌方向反轉。
  - 抽二 (Draw Two / +2)：下一位玩家抽 2 張且被跳過。
- **萬用牌（無花色）**：
  - 變色 (Wild)：出牌者指定接下來的花色。
  - 變色抽四 (Wild Draw Four / +4)：指定花色，且下一位玩家抽 4 張並被跳過。

### 3.2 出牌合法性規則
- 打出的牌必須符合下列任一條件才合法：
  1. 與棄牌堆頂牌**同花色**；
  2. 與頂牌**同數字**；
  3. 與頂牌**同功能**（如 Skip 疊 Skip）；
  4. 為**萬用牌**（Wild / Wild+4，任何情況皆可出，**[假設]** +4 可能有「手中無同色牌才可出」之限制）。
- 若無合法牌可出，玩家須從牌堆抽 1 張；**[假設]** 抽到可出的牌可選擇立即出或保留。

### 3.3 勝負判定
- 最先將手牌全部出完的玩家獲勝，該局結束。
- **[假設]** 可能有計分制（其餘玩家手牌點數加總給勝者），達到目標分數者為最終贏家。

### 3.4 UNO 宣告與罰則
- 當玩家打出倒數第二張牌、手牌剩 1 張時，須宣告「UNO」。
- **[假設]** 若未宣告而被其他玩家抓到，需罰抽牌（常見為 2 張）。

## 4. 功能需求（Functional Requirements）

### 4.1 使用者身分與進入
- **FR-1.1** 使用者可設定顯示暱稱後進入遊戲。**[假設]** 可能為訪客即玩（免註冊）或需登入。
- **FR-1.2** **[假設]** 支援帳號註冊 / 登入（Email 或第三方）。
- **FR-1.3** 暱稱應有長度與字元限制（Phase 3 確認）。

### 4.2 大廳與房間管理
- **FR-2.1** 使用者可**建立房間**，並取得可分享的房號 / 連結。
- **FR-2.2** 使用者可**加入房間**：透過房號、連結，或從房間清單點選。
- **FR-2.3** 房間有**人數上限**（**[假設]** 2–4 或 2–10 人）與**最低開局人數**（**[假設]** 至少 2 人）。
- **FR-2.4** 房主可設定房間參數（**[假設]** 私人/公開、密碼、AI 補位、規則變體）。
- **FR-2.5** 房主可**開始遊戲**；達最低人數才可開始。
- **FR-2.6** 玩家可**離開房間**；房主離開時房間應有轉移或關閉機制。
- **FR-2.7** **[假設]** 可加入 AI/電腦玩家補足人數。

### 4.3 遊戲進行
- **FR-3.1** 開局時系統為每位玩家發初始手牌（**[假設]** 7 張），並翻開一張作為棄牌堆頂牌。
- **FR-3.2** 系統顯示：自己的手牌、各對手剩餘張數、棄牌堆頂牌、當前花色、出牌方向、輪到誰。
- **FR-3.3** 輪到自己時，可點選**合法**手牌打出；不合法的牌應被拒絕或不可點選。
- **FR-3.4** 打出萬用牌時，須跳出**選色介面**指定花色。
- **FR-3.5** 無牌可出時可點**抽牌**；抽牌後依規則換手或可續出。
- **FR-3.6** 功能牌效果即時生效並同步給所有玩家（跳過、反轉、+2、+4）。
- **FR-3.7** 手牌剩 1 張時提供 **UNO 宣告**操作。
- **FR-3.8** 有玩家出完手牌時，顯示**結算/勝利畫面**。
- **FR-3.9** 局結束後可**再來一局 / 返回大廳**。

### 4.4 即時同步與連線
- **FR-4.1** 所有玩家看到的遊戲狀態應即時一致（出牌、抽牌、換手即時反映）。
- **FR-4.2** 玩家**斷線 / 重整**後應能重連回到當前對局。**[假設]** 有斷線重連或 AI 託管機制。
- **FR-4.3** **[假設]** 玩家離場過久會被判超時（自動抽牌換手或踢出）。

### 4.5 互動與輔助
- **FR-5.1** **[假設]** 房內聊天室 / 表情。
- **FR-5.2** **[假設]** 遊戲操作紀錄 / 出牌歷史。
- **FR-5.3** 提示當前非法操作的原因（如「此牌不可出」）。

## 5. 非功能需求（Non-Functional Requirements）

- **NFR-1 效能**：出牌到其他玩家看到更新的延遲應在可接受範圍（**[假設]** < 1 秒）。
- **NFR-2 相容性**：支援主流桌面瀏覽器（Chrome/Firefox/Safari/Edge）；**[假設]** 響應式支援手機。
- **NFR-3 可用性**：介面清楚指示「輪到你」、可出的牌、當前花色。
- **NFR-4 穩定性**：多人同時操作、網路抖動下不應造成狀態錯亂或卡死。
- **NFR-5 安全性**：玩家不能看到他人手牌；不能在非自己回合出牌；不能出非法牌；前端竄改請求應被伺服器端拒絕。
- **NFR-6 公平性**：發牌與洗牌應隨機、不可預測。
- **NFR-7 無障礙**：**[假設]** 色弱友善（花色不僅靠顏色區分）、鍵盤操作。

## 6. 使用者流程（User Flows）

### 6.1 主流程：建立房間並對戰
1. 進入首頁 → 設定暱稱
2. 建立房間 → 取得房號/連結
3. 等待其他玩家加入（或加 AI）
4. 房主開始遊戲 → 發牌
5. 依回合輪流出牌 / 抽牌 / 用功能牌 / 宣告 UNO
6. 有人出完 → 結算畫面
7. 再來一局 或 返回大廳

### 6.2 次流程：加入既有房間
1. 進入首頁 → 設定暱稱
2. 輸入房號 / 開啟邀請連結 / 從清單點選
3. 進入房間等待區 → 遊戲開始後參與對戰

## 7. 邊界與例外情境（黑箱推定）

- 房間已滿仍嘗試加入 → 應被拒並提示。
- 輸入不存在的房號 → 應提示查無房間。
- 人數不足硬要開局 → 應被阻止並提示。
- 非自己回合點手牌 → 應無效並提示。
- 打出不合法的牌 → 應被拒。
- 牌堆抽完 → **[假設]** 將棄牌堆（除頂牌）洗回牌堆。
- 對局中房主離開 → **[假設]** 房主轉移或以 AI 接管。
- 網路斷線 → 重連機制。
- 同時最後一張與對手出牌競態 → 伺服器仲裁順序。
