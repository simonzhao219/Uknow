# 路由順序錯誤修復報告

**修復日期：** 2024-12-15  
**問題類型：** 後端路由順序錯誤  
**嚴重程度：** 🔴 高（導致新功能完全無法使用）

---

## 📋 問題摘要

**錯誤表現：**
```
點擊「查看月度進度」→ ❌ 獲取月度摘要錯誤: ApiError: 無效的任務 ID
點擊「查看詳情」→ ❌ 獲取本月推薦錯誤: ApiError: 無效的任務 ID
```

**受影響功能：**
- ❌ 任務中心 - 連續推薦達人 - 查看月度進度（完全無法使用）
- ❌ 任務中心 - 推薦王 - 查看本月推薦（完全無法使用）
- ✅ 任務中心 - 其他功能（正常）

---

## 🐛 根本原因分析

### **問題根源：路由定義順序錯誤**

**錯誤的路由順序：**
```typescript
// /supabase/functions/server/tasks.ts

tasks.get('/', ...)                    // 1. GET /tasks/
tasks.get('/:taskId', ...)             // 2. GET /tasks/:taskId ❌ 動態路由在前
tasks.get('/details/:month', ...)      // 3. GET /tasks/details/:month ❌ 永遠不會執行
tasks.get('/monthly-summary', ...)     // 4. GET /tasks/monthly-summary ❌ 永遠不會執行
tasks.get('/current-month-top', ...)   // 5. GET /tasks/current-month-top ❌ 永遠不會執行
```

### **為什麼會這樣？**

**Hono 框架的路由匹配規則：**
1. **按定義順序匹配**：從上到下逐一檢查
2. **第一個匹配的路由生效**：找到匹配後立即執行，不再繼續
3. **動態路由匹配任何值**：`/:taskId` 會匹配任何路徑段

**實際發生過程：**

```
用戶請求：GET /tasks/monthly-summary

路由匹配過程：
1. 檢查 tasks.get('/')                 → ❌ 不匹配（路徑不是 /tasks/）
2. 檢查 tasks.get('/:taskId')          → ✅ 匹配！（taskId = "monthly-summary"）
3. 停止匹配，執行 /:taskId 的處理函數
4. /:taskId 處理函數檢查 taskId 值
5. if (taskId === 'consecutive_referral') → ❌ 不匹配
6. else if (taskId === 'monthly_king')    → ❌ 不匹配
7. else → 返回錯誤：「無效的任務 ID」

結果：/monthly-summary 永遠不會被執行
```

### **動態路由的匹配範圍**

```typescript
// /:taskId 會匹配以下所有路徑：
GET /tasks/consecutive_referral   ✅ 匹配（預期行為）
GET /tasks/monthly_king            ✅ 匹配（預期行為）
GET /tasks/monthly-summary         ✅ 匹配（意外！應該由固定路由處理）
GET /tasks/current-month-top       ✅ 匹配（意外！應該由固定路由處理）
GET /tasks/details/2024-12         ✅ 匹配（意外！應該由固定路由處理）
GET /tasks/any-random-string       ✅ 匹配（預期行為 - 返回錯誤）
```

---

## 🔧 修復方案

### **正確的路由順序**

**黃金規則：具體路由必須放在動態路由之前！**

```typescript
// ✅ 正確順序（從具體到動態）

tasks.get('/', ...)                    // 1. GET /tasks/ - 根路由
tasks.get('/details/:month', ...)      // 2. GET /tasks/details/:month - 具體路由
tasks.get('/monthly-summary', ...)     // 3. GET /tasks/monthly-summary - 具體路由
tasks.get('/current-month-top', ...)   // 4. GET /tasks/current-month-top - 具體路由
tasks.get('/:taskId', ...)             // 5. GET /tasks/:taskId - 動態路由（最後！）
```

### **修復後的匹配過程**

```
用戶請求：GET /tasks/monthly-summary

路由匹配過程：
1. 檢查 tasks.get('/')                 → ❌ 不匹配
2. 檢查 tasks.get('/details/:month')   → ❌ 不匹配
3. 檢查 tasks.get('/monthly-summary')  → ✅ 匹配！
4. 停止匹配，執行 /monthly-summary 的處理函數
5. 返回正確的數據

結果：路由正常工作
```
