# Pull Request 審查清單 - 獎勵與任務系統

## 使用說明

本清單用於審查所有與「獎勵回饋系統」和「任務系統」相關的 Pull Request。
請在提交 PR 時自行檢查，審查者也應逐項驗證。

---

## 通用檢查

- [ ] 是否遵循三時點架構（創建時/定時任務/讀取時）？
- [ ] 是否使用預計算快取而非即時計算？
- [ ] 是否更新相關的架構文檔（`/docs/REWARDS_TASKS_ARCHITECTURE.md`）？
- [ ] 是否新增或更新單元測試？
- [ ] 是否通過所有現有測試？
- [ ] 代碼是否有充分的錯誤處理？
- [ ] 是否有足夠的日誌記錄（console.log）？

---

## 推薦管理修改

### 資料讀取
- [ ] 是否使用 `listing:{listingId}:referral_tree` 讀取推薦樹？
- [ ] 是否使用 `user:{userId}:referral_stats` 讀取統計資料？
- [ ] 是否避免使用 `kv.getByPrefix('listing:')` 掃描所有刊登？

### 資料更新
- [ ] 刊登創建時是否更新 `referral_tree`？
- [ ] 刊登創建時是否更新 `referral_stats`？
- [ ] 是否正確計算推薦代數（最多3代）？

### 效能
- [ ] API 響應時間是否符合 < 100ms 要求？
- [ ] 是否避免 N+1 查詢問題？

---

## 獎勵回饋實作

### 刊登創建時（時點A）
- [ ] 是否調用 `processReferralRewards()` 函數？
- [ ] 是否立即發放第1個月獎勵？
- [ ] 是否創建11個獎勵排程（第2~12個月）？
- [ ] 獎勵排程是否加入日期索引（`reward_schedules_by_date:{date}`）？

### 獎勵資料
- [ ] 是否更新 `user:{userId}:rewards` 的所有欄位？
  - [ ] `availableRewards`
  - [ ] `totalEarned`
  - [ ] `lastUpdated`
- [ ] 是否記錄獎勵歷史（`user:{userId}:reward_history`）？
- [ ] 獎勵歷史是否限制在200筆以內？
- [ ] 是否記錄 `sourceUserName`（用於前端顯示）？
- [ ] 獎勵描述是否清晰易懂？

### 獎勵排程
- [ ] 排程是否包含所有必要欄位？
  - [ ] `userId`
  - [ ] `sourceListingId`
  - [ ] `sourceUserId`
  - [ ] `sourceUserName`
  - [ ] `generation`
  - [ ] `monthNumber`
  - [ ] `amount`
  - [ ] `scheduledDate`
  - [ ] `status`
- [ ] 日期計算是否正確（每月同日 00:00）？
- [ ] 是否為每個排程生成唯一 ID？

### 資料一致性
- [ ] `totalEarned` 是否正確計算？
- [ ] 獎勵金額是否符合 spec（每月10P）？
- [ ] 是否正確處理推薦代數（1代、2代、3代）？

---

## 任務系統實作

### 任務進度追蹤
- [ ] 是否正確更新 `user:{userId}:referral_monthly_log`？
- [ ] 月度日誌是否只記錄第1代推薦？
- [ ] 是否使用正確的月份格式（"YYYY-MM"）？

### 連續推薦任務
- [ ] 是否檢查連續性（上個月是否有推薦）？
- [ ] 斷續時是否正確重置任務（`currentStreak` 歸零）？
- [ ] 是否正確記錄 `monthlyRecord`？
- [ ] 達到12個月時是否標記為 `completed`？
- [ ] 完成後是否重置開始新一輪？
- [ ] 是否記錄 `lastCheckedAt` 時間戳？

### 推薦王任務
- [ ] 是否正確統計本月推薦數？
- [ ] 換月時是否正確重置計數器？
- [ ] 上個月記錄是否加入 `history`？
- [ ] 達到10個時是否標記為 `completed`？
- [ ] 是否防止重複發放獎勵？

### 任務獎勵
- [ ] 任務完成時是否調用 `issueTaskReward()`？
- [ ] 獎勵金額是否符合 spec（1000P）？
- [ ] 是否記錄獎勵歷史？
- [ ] `type` 是否正確標記為 `task_{taskType}`？

## 定時任務

### GitHub Actions 配置
- [ ] workflow 檔案是否存在（`.github/workflows/daily-rewards-cron.yml`）？
- [ ] Cron 時間是否為 UTC 16:05（台北00:05）？
- [ ] 是否設定正確的 secrets？
  - [ ] `SUPABASE_PROJECT_ID`
  - [ ] `SUPABASE_SERVICE_ROLE_KEY`
- [ ] 是否支援手動觸發（`workflow_dispatch`）？
- [ ] 失敗時是否有通知機制？

### API 端點（/cron/process-daily-rewards）
- [ ] 是否驗證 Service Role Key？
- [ ] 是否拒絕未授權的請求？
- [ ] 是否記錄開始和結束時間？
- [ ] 是否返回處理統計資料？

### 推薦獎勵處理
- [ ] 是否使用日期索引快速查找排程？
- [ ] 是否檢查來源刊登的 `cancelledAt` 狀態？
- [ ] 是否檢查來源刊登的 `activeUntil` 有效期？
- [ ] 刊登無效時是否將排程標記為 `cancelled`？
- [ ] 發放成功時是否將排程標記為 `completed`？
- [ ] 是否記錄 `completedAt` 時間戳？
- [ ] 是否正確更新用戶獎勵餘額？
- [ ] 是否記錄獎勵歷史？

### 任務結算
- [ ] 是否只在每月1日執行？
- [ ] 是否正確計算上個月份（"YYYY-MM"）？
- [ ] 連續推薦任務是否正確判斷斷續？
- [ ] 推薦王任務是否正確判斷達標？
- [ ] 任務完成時是否發放獎勵？
- [ ] 是否防止重複發放（`completed` 標記）？
- [ ] 是否更新 `lastCheckedAt` 時間戳？

### 錯誤處理
- [ ] 是否有 try-catch 包裹主要邏輯？
- [ ] 錯誤是否記錄到日誌？
- [ ] 部分失敗是否繼續處理其他項目？
- [ ] 是否返回詳細的錯誤訊息？

## 前端整合

### 獎勵回饋頁面
- [ ] 是否調用 `GET /rewards` API？
- [ ] 是否正確顯示所有獎勵資料？
  - [ ] 可提領點數（`availableRewards`）
  - [ ] 處理中點數（`pendingRewards`）
  - [ ] 已提領點數（`withdrawnRewards`）
  - [ ] 總累積點數（`totalEarned`）
- [ ] 是否調用 `GET /rewards/history` API？
- [ ] 獎勵歷史是否正確顯示？
- [ ] 是否有載入狀態（loading）？
- [ ] 是否有錯誤處理？

### 任務系統頁面
- [ ] 是否調用 `GET /tasks` API？
- [ ] 是否正確顯示連續推薦任務？
  - [ ] 任務標題和描述
  - [ ] 當前進度（x / 12 個月）
  - [ ] 進度條
  - [ ] 獎勵金額
- [ ] 是否正確顯示推薦王任務？
  - [ ] 任務標題和描述
  - [ ] 當前進度（x / 10 個推薦）
  - [ ] 進度條
  - [ ] 獎勵金額
  - [ ] 當前月份
- [ ] 是否有載入狀態（loading）？
- [ ] 是否有錯誤處理？

### 推薦管理頁面
- [ ] API 響應時間是否明顯改善？
- [ ] 資料是否正確顯示？
- [ ] 是否有任何功能退化？

## 效能驗證

### 響應時間
- [ ] 刊登創建是否在 2 秒內完成（包含獎勵處理）？
- [ ] 推薦管理 API 是否在 100ms 內響應？
- [ ] 獎勵回饋 API 是否在 50ms 內響應？
- [ ] 任務系統 API 是否在 50ms 內響應？
- [ ] 定時任務是否在 5 分鐘內處理完所有用戶？

### 效能測試
- [ ] 是否進行壓力測試（100+ 個用戶）？
- [ ] 是否測試 1000+ 個刊登的情況？
- [ ] 是否有效能監控（console.time/timeEnd）？

---

## 資料一致性

### 獎勵資料
- [ ] `totalEarned` 是否等於 `availableRewards + pendingRewards + withdrawnRewards`？
- [ ] 獎勵歷史總額是否與 `totalEarned` 一致？
- [ ] 所有獎勵記錄是否有完整資訊？

### 推薦資料
- [ ] `totalReferrals` 是否等於 `firstGenCount + secondGenCount + thirdGenCount`？
- [ ] 推薦樹中的各代數量是否與統計資料一致？

### 任務資料
- [ ] 連續推薦月數是否與 `monthlyRecord` 長度一致？
- [ ] 推薦王的本月計數是否與月度日誌一致？

- [ ] 是否處理來源刊登不存在的情況？
